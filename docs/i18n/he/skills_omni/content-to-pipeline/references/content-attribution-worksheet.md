Use this worksheet to create a lightweight but repeatable content attribution system.

## 1. UTM naming convention

Use a consistent pattern across all content links.

| Field | Purpose | Example |
| --- | --- | --- |
| `utm_source` | platform or origin | `linkedin`, `youtube`, `newsletter` |
| `utm_medium` | distribution type | `organic_social`, `email`, `partner` |
| `utm_campaign` | campaign or content series | `q2_ops_playbook` |
| `utm_content` | specific asset or variation | `carousel_01`, `video_hook_a` |

Rules:

- keep values lowercase
- use hyphens or underscores consistently
- avoid ad hoc abbreviations no one will understand later
- document campaign names before publishing

## 2. Minimum conversion events

Define the events that matter before scaling content.

| Event | Why it matters | Typical owner |
| --- | --- | --- |
| newsletter signup | owned audience capture | marketing or founder |
| lead magnet download | problem-aware conversion | marketing |
| demo request | direct pipeline intent | sales or founder |
| contact form submit | hand-raise interest | sales or ops |
| booked meeting | qualified conversation | sales |

## 3. CRM properties to capture

At minimum, capture:

- original source
- latest source
- campaign name if available
- self-reported attribution
- content topic or content asset note when known
- lifecycle stage
- opportunity created date

## 4. Self-reported attribution

Add one required or strongly encouraged form field:

- How did you hear about us?

If space allows, add a second optional field:

- What content, person, or channel influenced you most?

Also add a sales-call prompt:

- Before we dive in, what content of ours, if any, have you already seen?

## 5. Monthly review questions

Review once per month:

1. Which content themes produced the most qualified responses?
2. Which channels created the most owned audience growth?
3. Which offers converted subscribers or engaged prospects into meetings?
4. Which deals mention content even when software attribution is missing?
5. Which naming or capture failures made reporting unreliable?

## 6. Interpretation guardrails

- Treat software attribution and self-report as complementary.
- Expect undercounting where consent, privacy, or dark social are involved.
- Do not claim exact causality when multiple touches influenced the deal.
- Use directional patterns to improve decisions, not to pretend measurement is perfect.

Use this worksheet to compare AI SDR options without over-trusting vendor marketing.

## 1. Decision categories

Score each option on:

- autonomy level
- approval controls
- CRM fit
- deliverability ownership clarity
- enrichment/data fit
- reporting quality
- operator workload
- pilot complexity
- cost structure
- exit risk / portability

## 2. Questions to ask

### Workflow fit

- What part of the SDR workflow is actually automated?
- What still requires human review?
- Can the operator restrict risky reply categories?

### Controls

- Can pricing, legal, or security replies be blocked from autonomous handling?
- Is there an approval layer for sensitive responses?
- Can suppression and do-not-contact states be enforced reliably?

### CRM and routing

- How are owners assigned?
- How are lifecycle stages updated?
- How are duplicates prevented?

### Deliverability and sending

- Who owns mailbox/domain configuration?
- How are complaints, bounces, and unsubscribe events surfaced?
- Does the tool encourage risky scaling behavior?

### Evaluation

- Can the team review labeled reply outcomes?
- Can routing performance be audited?
- Can meeting quality be tied back to qualification logic?

## 3. Warning signs

Be cautious if a vendor:

- promises full autonomy with little operator oversight
- does not explain sensitive-reply controls
- cannot show how suppression is enforced
- focuses on volume without sender health discussion
- makes ROI claims without evidence requirements

## 4. Output format

A useful comparison output includes:

- top 3 options
- best-fit scenario for each
- top risks for each
- minimum pilot plan required before broader rollout

## 5. Reminder

Vendor choice rarely fixes weak ICP, poor data quality, unclear routing, or bad handoff design.

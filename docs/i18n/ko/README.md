## 빠른 시작

> 🔍**AI 코딩 기술, Claude Code 기술, 커서 기술, Codex CLI 기술, Gemini CLI 기술, 반중력 기술 또는 설치 가능한 `SKILL.md` 라이브러리를 검색하셨나요?**
>**잘 찾아오셨습니다.**👇### 1️⃣ What is this?

Awesome Omni Skills는 AI 코딩 도우미를 위한**설치 가능한 스킬 카탈로그 및 런타임**입니다. 핵심은 재사용 가능한 'SKILL.md' 플레이북의 공개 저장소, 개선된 모범 사례 파생물의 엄선된 저장소, 동일한 카탈로그를 배포하고 운영하는 CLI, API, MCP 및 A2A 런타임 표면입니다.

기본 섭취는 직접 기여자 PR과 검토된 저장소 기반 외부 가져오기 PR이라는 두 가지 방법으로 'skills/'에 입력됩니다. 그런 다음 선별된 영어 전용 파생물은 개인 향상 파이프라인에 의해 `skills_omni/`에 별도로 제안됩니다.

## 🧩 Core Concepts

번들을 비교하거나 설치 경로를 선택하기 전에 다음 5가지 구성 요소를 이해하면 도움이 됩니다.

| 개념 | 의미 |
|:---------|:-------------|
| 🧠**기술**| 어시스턴트에게 워크플로를 효과적으로 실행하는 방법을 가르치는 재사용 가능한 `SKILL.md` 플레이북 |
| 📦**카탈로그 아티팩트**| 검색, 비교, 다운로드 및 설치가 가능한 생성된 JSON 및 아카이브 출력 |
| 🔌**MCP 구성**| 보조자가 MCP 도구를 통해 Awesome Omni Skills를 발견할 수 있도록 하는 클라이언트측 구성 |
| 🤖**A2A 런타임**| 검색, 추천 및 설치 계획 전달을 위한 에이전트 간 오케스트레이션 |
| ✨**선별된 결과물**| `skills_omni/` — 기본 업스트림 흡입구와 별도로 Awesome Omni Skills로 유지되는 향상된 표면 |

>**📝 네이티브/선별 정책:**
> - `skills/`는 직접 제출했든 비공개 외부 동기화 런타임에서 제안했든 상관없이 모든 언어의 기본 업스트림 섭취를 허용합니다.
> - `skills_omni/`는 항상 영어로 큐레이팅되어 게시됩니다.
> - `skills_omni/`는 단방향 표면이며 기본 섭취로 다시 루프되지 않습니다.
> - 직접 기여자 PR과 저장소 기반 외부 섭취는 동일한 유효성 검사기, 메타데이터 새로 고침 및 강화 경로를 공유합니다.---

- 네이티브 인테이크는 중요한 보안 결과가 있는 기술을 허용하지 않습니다.
- 정적 유효성 검사기는 기술이 공개 카탈로그에 병합되기 전에 고위험 패턴을 차단합니다.
- 현재 하드 차단 프로그램에는 다음이 포함됩니다.
  - 쉘 실행으로 직접 파이프되는 원격 콘텐츠
  - 프롬프트, 비밀 또는 숨겨진 런타임 컨텍스트를 공개하려는 지침
- 게시된 릴리스는 더 엄격한 두 번째 계층을 거칩니다.
  - ClamAV 스캐닝
  - VirusTotal 해시 조회
  - 아카이브 체크섬 확인
  - 릴리스 CI의 분리된 서명 확인

이는 배관 구축뿐만 아니라 사용자가 직면하는 이점입니다. 카탈로그는 프로젝트가 내부적으로 사용하는 것과 동일한 제한적인 보안 게이트를 이미 통과한 설치 가능한 기술을 선호합니다.</details>

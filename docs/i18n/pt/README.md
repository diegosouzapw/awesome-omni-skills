## Início Rápido

> 🔍**Procurou habilidades de codificação de IA, habilidades de Claude Code, habilidades de cursor, habilidades Codex CLI, habilidades Gemini CLI, habilidades antigravidade ou bibliotecas `SKILL.md` instaláveis?**
>**Você está no lugar certo.**👇### 1️⃣ What is this?

Awesome Omni Skills é um**catálogo de habilidades instaláveis ​​e tempo de execução**para assistentes de codificação de IA. Em sua essência, é um repositório público de manuais `SKILL.md` reutilizáveis, um repositório com curadoria de derivados de melhores práticas aprimorados e superfícies de tempo de execução CLI, API, MCP e A2A que distribuem e operam o mesmo catálogo.

A ingestão nativa entra em `habilidades/` de duas maneiras: PRs de contribuidores diretos e PRs de importação externa baseados em repositórios revisados. Derivados selecionados apenas em inglês são então propostos separadamente em `skills_omni/` pelo pipeline de aprimoramento privado.

| Componente | Descrição |
|:----------|:-----------|
| 🧠**Habilidades**| Manuais reutilizáveis ​​baseados em `SKILL.md` para assistentes de IA, incluindo ingestão nativa de PRs de contribuidores diretos e PRs de importação externa baseados em repositório revisados ​​|
| 📦**Manifestos**| Manifestos, pacotes e arquivos JSON gerados |

## 🧩 Core Concepts

Antes de comparar pacotes ou escolher um caminho de instalação, compreender estes cinco blocos de construção ajuda:

| Conceito | O que significa |
|:--------|:-------------|
| 🧠**Habilidades**| Manuais `SKILL.md` reutilizáveis ​​que ensinam um assistente como executar bem um fluxo de trabalho |
| 📦**Artefatos do Catálogo**| JSON gerado e saídas de arquivo permitindo pesquisa, comparação, download e instalação |
| 🔌**Configuração MCP**| Configuração do lado do cliente para assistentes descobrirem Awesome Omni Skills por meio de ferramentas MCP |
| 🤖**Tempo de execução A2A**| Orquestração entre agentes para descoberta, recomendação e transferência de plano de instalação |
| ✨**Saída selecionada**| `skills_omni/` — a superfície aprimorada mantida pelo Awesome Omni Skills, separada da entrada upstream nativa |

>**📝 Política nativa/com curadoria:**
> - `skills/` aceita entrada de upstream nativo em qualquer idioma, seja enviado diretamente ou proposto pelo tempo de execução de sincronização externa privada
> - `skills_omni/` é sempre curado e publicado em inglês
> - `skills_omni/` é uma superfície unidirecional e não retorna à ingestão nativa
> - PRs de contribuidores diretos e entrada externa baseada em repositório compartilham o mesmo validador, atualização de metadados e caminho de aprimoramento---

## 🛡️ Security and Release Posture

> Awesome Omni Skills oferece uma história de lançamento e verificação mais forte do que um repositório de descontos simples.### 🧪 Validation and Smoke Checks

```bash
npm run validate         # Skill validation and metadata generation
npm run build            # Full build pipeline
npm test                 # Legacy integration tests + Vitest unit suite
npm run test:unit        # Fast Vitest unit tests only (~500ms)
npm run test:coverage    # Vitest with V8 coverage
npm run smoke            # Full release preflight
```

<detalhes>
<summary>📋 <strong>O que o pipeline valida</strong></summary>

- ✅ Validação de habilidades e geração de metadados
- ✅ Controle crítico de segurança na entrada nativa antes da fusão de relações públicas
- ✅ Ferramentas de normalização e recategorização de taxonomia
- ✅ Geração de catálogo e arquivo
- ✅ Testes de integração legados (afirmações Python PTY + Node TUI)
- ✅ Conjunto de unidades Vitest para pontuação, pesquisa e lógica de filtragem do núcleo do catálogo
- ✅ Caminhos de inicialização API, MCP e A2A
- ✅ Verificação de arquivo
- ✅ Contrato OpenAPI 3.1 servido via Swagger UI em `/docs`
- ✅ Pré-comprovação de pacote com `npm pack --dry-run`</details>

<detalhes>
<summary>🔐 <strong>Postura de liberação</strong></summary>

| Controle | Descrição |
|:--------|:-----------|
| 🔒 Somas de verificação SHA-256 | Manifestos de soma de verificação para todos os arquivos |
| ✍️ Artefatos assinados | Assinaturas separadas em artefatos de lançamento |
| 🤖 Aplicado por CI | Verificação de liberação no CI antes da publicação |
| 🦠 Portões com scanner | ClamAV e VirusTotal são necessários na verificação de lançamento antes da publicação |
| 📦 Lançamento do GitHub | Geração automatizada de lançamento do GitHub |
| 📋 publicação npm | Apenas do tarball verificado |
| 🔄 Liberação automática | Na qualificação, a habilidade é mesclada com `principal` |

**A liberação automática é acionada apenas quando uma mesclagem é alterada:**
- `habilidades/*/**`
- `skills_omni/*/**`
- `dados/bundles.json`

Alterações somente em documentos**não**acionam a publicação do pacote.</details>

<detalhes>
<summary>🛡️ <strong>Contrato de segurança do repositório</strong></summary>

- A ingestão nativa não aceita habilidades com resultados críticos de segurança.
- O validador estático bloqueia padrões de alto risco antes que uma habilidade possa ser mesclada no catálogo público.
- Os bloqueadores rígidos atuais incluem:
  - conteúdo remoto canalizado diretamente para a execução do shell
  - instruções que tentam revelar prompts, segredos ou contexto de tempo de execução oculto
- Os lançamentos publicados passam por uma segunda camada mais rigorosa:
  - Varredura ClamAV
  - Pesquisa de hash do VirusTotal
  - verificação de soma de verificação de arquivo
  - verificação de assinatura desanexada no lançamento CI

Este é um benefício voltado para o usuário, não apenas para a construção de encanamentos: o catálogo favorece habilidades instaláveis que já passaram pelo mesmo portão de segurança restritivo que o projeto usa internamente.</details>

## Contribuir

Awesome Omni Skills aceita o consumo de habilidades upstream nativas em `skills/`.

| Regra | Detalhes |
|:-----|:--------|
| 📥 Ingestão nativa | Pode ser grosseiro, escrito em qualquer idioma e pode chegar de PRs diretos ou de filiais de importação externa revisadas |
| ✨ Produção selecionada | `skills_omni/` reservado para derivados Omni de autoria de automação |
| 🌍 Propostas de recompra upstream | Adicione ou atualize uma linha em [REPOSITORY-SOURCES.md](REPOSITORY-SOURCES.md) para nomear um repositório para revisão do operador; propostas repo-first agora podem usar `branch=auto` e `skills_path=auto` |
| 🚫 Edições manuais | Edições manuais públicas em `skills_omni/` são rejeitadas |
| 🔄 Reprocessamento | O aprimorador privado reprocessa alterações nativas e atualiza a linha de base selecionada |

> 📖**Comece com:**[Guia de contribuição](CONTRIBUTING.md) · [Fluxo de trabalho de RP de habilidades](docs/contributors/SKILL-PR-WORKFLOW.md) · [Fontes de repositório](REPOSITORY-SOURCES.md)---

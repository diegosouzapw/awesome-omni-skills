## Быстрый старт

> 🔍**Ищете навыки кодирования AI, навыки Claude Code, навыки курсора, навыки CLI Codex, навыки Gemini CLI, навыки антигравитации или устанавливаемые библиотеки `SKILL.md`?**
>**Вы по адресу.**👇### 1️⃣ What is this?

Awesome Omni Skills — это**устанавливаемый каталог навыков и среда выполнения**для помощников по программированию с использованием искусственного интеллекта. По своей сути это общедоступный репозиторий многоразовых сборников сценариев `SKILL.md`, курируемый репозиторий улучшенных производных лучших практик, а также поверхности среды выполнения CLI, API, MCP и A2A, которые распространяют и управляют одним и тем же каталогом.

Собственный прием вводит `навыки/` двумя способами: PR прямого участника и проверенные PR внешнего импорта на основе репозитория. Курируемые производные версии только на английском языке затем предлагаются отдельно в `skills_omni/` с помощью частного конвейера усовершенствований.

## 🧩 Core Concepts

Прежде чем сравнивать пакеты или выбирать путь установки, полезно понять эти пять строительных блоков:

| Концепция | Что это значит |
|:--------|:-------------|
| 🧠**Навыки**| Многоразовые сборники инструкций `SKILL.md`, которые учат помощника тому, как хорошо выполнять рабочий процесс |
| 📦**Артефакты каталога**| Сгенерированные результаты JSON и архивы, позволяющие осуществлять поиск, сравнение, загрузку и установку |
| 🔌**Конфигурация MCP**| Конфигурация на стороне клиента, позволяющая ассистентам открывать для себя Awesome Omni Skills с помощью инструментов MCP |
| 🤖**Среда выполнения A2A**| Межагентная оркестровка для обнаружения, рекомендаций и передачи плана установки |
| ✨**Рекомендуемый результат**| `skills_omni/` — улучшенная поверхность, поддерживаемая Awesome Omni Skills, отдельная от исходного входного потока |

>**📝 Собственная/курируемая политика:**
> - `skills/` принимает встроенный исходный ввод на любом языке, независимо от того, отправлен ли он напрямую или предложен с помощью частной среды выполнения внешней синхронизации.
> - `skills_omni/` всегда курируется и публикуется на английском языке.
> - `skills_omni/` является односторонней поверхностью и не возвращается к собственному вводу
> - PR для прямых участников и внешний прием на основе репозитория используют один и тот же путь проверки, обновления метаданных и расширения.---

## 🔌 Runtime Surfaces

Awesome Omni Skills — это не только библиотека навыков. Он предоставляет**четыре поверхности времени выполнения**, которые используют один и тот же сгенерированный каталог:

| Поверхность | Государство | Что он делает | Пример |
|:--------|:------|:-------------|:--------|
| 🖥️**CLI**| ✅ Доступен | Поиск, установка, диагностика, визуальный пользовательский интерфейс, службы загрузки, дымовые проверки | `npx доктор удивительных омни-навыков` |
| 🌐**API каталога**| ✅ Доступен | Каталог только для чтения, поиск, пакеты, сравнение, планы установки, загрузки, пользовательский интерфейс Swagger в `/docs` | `npx Awesome-OMNI-Skills API --port 3333` |
| 🔌**MCP**| ✅ Доступен | Обнаружение, рекомендации, предварительная установка, локальная коляска, потоки настройки | `npx Awesome-omni-Skills mcp поток --local` |
| 🤖**A2A**| ✅ Доступен | Жизненный цикл задачи, передача обслуживания, опрос, потоковая передача, отмена, сохранение | `npx Awesome-omni-skills a2a --port 3335` |

**Автоматическое освобождение срабатывает только при изменении слияния:**
- `навыки/*/**`
- `skills_omni/*/**`
- `данные/bundles.json`

Изменения только в документации**не**вызывают публикацию пакета.</details>

<подробности>
<summary>🛡️ <strong>Договор о безопасности репозитория</strong></summary>

- Нативный прием не принимает навыки с критическими показателями безопасности.
- Статический валидатор блокирует шаблоны высокого риска, прежде чем навык сможет быть добавлен в общедоступный каталог.
- Текущие жесткие блокировщики включают в себя:
  - удаленный контент передается непосредственно в выполнение оболочки
  - инструкции, которые пытаются раскрыть подсказки, секреты или скрытый контекст выполнения.
- Опубликованные релизы проходят более строгий второй уровень:
  - Сканирование ClamAV
  - Поиск по хешу VirusTotal
  - проверка контрольной суммы архива
  - отдельная проверка подписи в релизе CI

Это преимущество, ориентированное на пользователя, а не только на сборку сантехники: в каталоге отдаются предпочтение устанавливаемым навыкам, которые уже прошли те же самые ограничительные ворота безопасности, которые проект использует внутри себя.</details>

## 🗂️ Repository Layout

| Путь | Цель |
|:-----|:--------|
| 📂 `навыки/` | Канонические авторские навыки и собственное получение из прямых PR или проверенного импорта на основе репозитория |
| ✨ `skills_omni/` | Кураторские расширенные производные, поддерживаемые всеми пользователями |
| 📖 `документы/` | Документация по пользователям, участникам, архитектуре, операциям и спецификациям |
| 📦 `расстояние/` | Созданные манифесты, пакеты, каталог и архивы |
| 📁 `данные/` | Определения пакетов и статические вспомогательные данные |
| 🧠 `packages/catalog-core/` | Среда выполнения общего каталога с внедрением зависимостей `ICatalogStorageAdapter` |
| 🌐 `packages/server-api/` | HTTP API только для чтения с пользовательским интерфейсом OpenAPI/Swagger в `/docs` |
| 🔌 `packages/server-mcp/` | Сервер MCP и локальная коляска |
| 🤖 `packages/server-a2a/` | Среда выполнения A2A и оркестровка задач |

## Вклад

Awesome Omni Skills принимает встроенные исходные навыки в разделе `skills/`.

| Правило | Подробности |
|:-----|:--------|
| 📥Нативный прием | Может быть черновым, написанным на любом языке и полученным из прямых PR или проверенных ветвей внешнего импорта |
| ✨ Кураторский выпуск | `skills_omni/` зарезервировано для производных Omni, созданных с помощью автоматизации |
| 🌍 Предложения РЕПО по разведке и добыче | Добавьте или обновите одну строку в [REPOSITORY-SOURCES.md](REPOSITORY-SOURCES.md), чтобы назначить репозиторий на рассмотрение оператора; предложения repo-first теперь могут использовать `branch=auto` и `skills_path=auto` |
| 🚫 Ручное редактирование | Публичные изменения, внесенные вручную в `skills_omni/`, отклонены |
| 🔄Переработка | Частный энхансер повторно обрабатывает собственные изменения и обновляет тщательно подобранную базовую версию |

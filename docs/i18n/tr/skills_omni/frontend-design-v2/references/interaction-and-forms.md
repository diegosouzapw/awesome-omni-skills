Production-grade design includes state design, semantics, and usable forms.

## Controls

- Use buttons for actions.
- Use links for navigation.
- Make hover, focus, active, disabled, and loading states intentional.
- Do not make every action look primary.

## Forms

For common fields, use suitable `autocomplete` tokens when supported.

Examples:
- `name`
- `email`
- `username`
- `street-address`
- `postal-code`

## Form Rules

- Keep visible labels.
- Place helper text where users need it.
- Make validation messages specific.
- Preserve keyboard access.
- Ensure target sizes are usable.

## State Design

Design all of these where relevant:
- empty state
- loading state
- success state
- error state
- disabled state
- inline validation state

## Avoid

- placeholder-only labeling
- hidden affordances with no visible cue
- controls with weak focus treatment
- repeating the same information in headers, body copy, and controls

This example shows how product and sales can coordinate without interrupting activation too early.

## Example trigger

An account becomes a PQL when all of the following are true:

- 5 or more active users from the same domain
- at least 2 repeat usage sessions in a 7-day window
- one admin or billing-intent action observed
- account matches the target segment

## Example exclusions

- student or personal email domains
- internal testing accounts
- duplicate accounts already owned by sales
- activity limited to sample data only

## Sales entry policy

Sales should not enter at the first sign of activity.

Sales enters only after the account has:

1. demonstrated actual product value
2. shown team-level adoption or admin intent
3. reached a point where human help can accelerate purchase or rollout

## First sales action

The first message should be narrow and useful. Example goals:

- offer rollout help for the team
- answer security or procurement questions
- explain admin or enterprise controls
- support expansion planning

## Bad handoff pattern

Do not hand off because a single user clicked around a pricing page.

## Success criteria

A good handoff should increase one of these outcomes:

- movement into a real evaluation process
- expansion of team rollout
- progress through procurement or security review
- improved conversion among high-fit accounts

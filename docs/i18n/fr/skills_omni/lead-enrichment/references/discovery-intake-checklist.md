Use this before proposing or revising an enrichment workflow.

## 1. Goal

- What problem is the workflow solving?
- Is success defined as coverage, verified contacts, scoring quality, routing quality, or pipeline impact?
- What downstream action depends on the output?

## 2. ICP definition

- Target industries
- Company size bands
- Geography / allowed regions
- Target personas / departments
- Disqualifiers
- Existing best-fit customer patterns from CRM

## 3. Required fields

Mark each field as `required`, `useful`, or `avoid`.

## 4. Operating constraints

- Monthly record volume
- Budget or credit limit
- Preferred tools/providers already in use
- CRM / MAP / outbound destinations
- Freshness requirements
- Existing suppression or do-not-contact sources

## 5. Compliance / handling checks

- Target regions with stricter constraints
- Opt-out or suppression data source
- Whether lawful basis or outreach eligibility needs escalation
- Retention expectation
- DSR / deletion expectations

## 6. Success metrics

- Fill rate target
- Verified coverage target
- Cost per verified lead target
- Bounce rate threshold
- Complaint rate threshold
- Re-enrichment cadence by field or segment

Route away from this skill when the task no longer centers on Astro implementation work.

## Route to `perf-lighthouse`

Use when the request is primarily:
- running Lighthouse
- reading audit reports
- setting budgets or CI thresholds

## Route to `core-web-vitals`

Use when the request is primarily:
- interpreting LCP, CLS, or INP trends
- prioritizing metric strategy across multiple stacks
- understanding field versus lab tradeoffs

## Route to `perf-web-optimization`

Use when the request is primarily:
- framework-agnostic frontend performance work
- caching, preload strategy, bundle size, or third-party guidance outside Astro specifics
- a non-Astro site

## Route to accessibility support

Use when changes may affect:
- consent and script timing
- motion or transitions
- font readability and text rendering
- interaction timing and usability

## Route to server or infrastructure specialists

Use when the dominant issue is:
- slow SSR response time
- origin or adapter latency
- caching and CDN behavior
- backend waterfalls or API latency

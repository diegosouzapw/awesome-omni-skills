Use this note when the request moves beyond pricing strategy.

## Stay in `ai-pricing` when

- the main question is packaging, tiers, usage model, margins, BYOK tradeoffs, or migration strategy
- the output is a decision memo, pricing recommendation, or review packet

## Hand off when

### Billing implementation becomes the main task
Route when the user now needs:
- metering implementation
- invoice logic
- entitlement design
- billing provider integration

### Product or architecture decisions dominate
Route when the user now needs:
- system architecture for model routing or caching
- observability implementation
- production cost instrumentation design

### Finance or legal ownership dominates
Route when the user now needs:
- board-level planning
- accounting treatment
- legal review of contracts, notices, or procurement language

## Handoff packet

When handing off, carry forward:
- product archetype
- buyer and sales motion
- recommended external billable metric
- internal cost meters used for analysis
- main margin risks
- migration and BYOK assumptions

Use this worksheet before starting the audit.

## Inventory

| Candidate | Type | Why it matters | Public or authenticated | Include? |
| --- | --- | --- | --- | --- |
| Homepage | Entry page | First impression, discovery, shared template | Public |  |
| Key template page | Category / article / feature / listing | Represents repeated layout or content pattern | Public |  |
| Conversion page | Pricing / sign-up / contact / checkout | Business-critical path | Public or authenticated |  |
| Stateful page | Dashboard / settings / account | Dynamic UX, JS-heavy interactions | Authenticated |  |
| Search or detail flow | Search -> result -> detail | Common user journey | Public or authenticated |  |

## Selection rules

- Prefer 3-5 representative pages or steps.
- Do not rely on the homepage alone if the site has varied templates or app states.
- Include at least one page users visit often and one page the business depends on.
- Include at least one dynamic or authenticated experience when relevant.

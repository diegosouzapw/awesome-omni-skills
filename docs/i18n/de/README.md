## Schnellstart

> 🔍**Gesucht nach KI-Codierungskenntnissen, Claude-Code-Kenntnissen, Cursor-Kenntnissen, Codex-CLI-Kenntnissen, Gemini-CLI-Kenntnissen, Antigravitations-Kenntnissen oder installierbaren „SKILL.md“-Bibliotheken?**
>**Hier sind Sie richtig.**👇### 1️⃣ What is this?

Awesome Omni Skills ist ein**installierbarer Kompetenzkatalog und eine Laufzeit**für KI-Codierungsassistenten. Im Kern handelt es sich um ein öffentliches Repository wiederverwendbarer „SKILL.md“-Playbooks, ein kuratiertes Repository verbesserter Best-Practice-Derivate sowie die CLI-, API-, MCP- und A2A-Laufzeitoberflächen, die denselben Katalog verteilen und betreiben.

Die native Aufnahme geht auf zwei Arten in „Fähigkeiten/“ ein: direkte PRs von Mitwirkenden und überprüfte, auf dem Repository basierende, externe Import-PRs. Kuratierte, nur englischsprachige Derivate werden dann von der privaten Erweiterungspipeline separat in „skills_omni/“ vorgeschlagen.

## 🧩 Core Concepts

Bevor Sie Bundles vergleichen oder einen Installationspfad auswählen, ist es hilfreich, diese fünf Bausteine zu verstehen:

| Konzept | Was es bedeutet |
|:--------|:-------------|
| 🧠**Fähigkeiten**| Wiederverwendbare „SKILL.md“-Playbooks, die einem Assistenten beibringen, wie man einen Workflow gut ausführt |
| 📦**Katalogartefakte**| Generierte JSON- und Archivausgaben, die Suche, Vergleich, Herunterladen und Installation ermöglichen |
| 🔌**MCP-Konfiguration**| Clientseitige Konfiguration für Assistenten, um fantastische Omni-Fähigkeiten mithilfe von MCP-Tools zu entdecken |
| 🤖**A2A-Laufzeit**| Agent-zu-Agent-Orchestrierung für Erkennung, Empfehlung und Installationsplanübergabe |
| ✨**Kuratierte Ausgabe**| „skills_omni/“ – die von Awesome Omni Skills gepflegte erweiterte Oberfläche, getrennt von der nativen Upstream-Einnahme |

>**📝 Native/kuratierte Richtlinie:**
> - „skills/“ akzeptiert native Upstream-Aufnahmen in jeder Sprache, unabhängig davon, ob diese direkt übermittelt oder von der privaten External-Sync-Laufzeit vorgeschlagen werden
> - „skills_omni/“ wird immer auf Englisch kuratiert und veröffentlicht
> - „skills_omni/“ ist eine Einwegoberfläche und führt nicht zurück zur nativen Aufnahme
> – PRs direkter Mitwirkender und Repository-basierte externe Aufnahme nutzen denselben Validierungs-, Metadatenaktualisierungs- und Enhancer-Pfad---

## 🛡️ Security and Release Posture

> Awesome Omni Skills liefert eine stärkere Release- und Verifizierungsgeschichte als ein einfaches Markdown-Repository.### 🧪 Validation and Smoke Checks

```bash
npm run validate         # Skill validation and metadata generation
npm run build            # Full build pipeline
npm test                 # Legacy integration tests + Vitest unit suite
npm run test:unit        # Fast Vitest unit tests only (~500ms)
npm run test:coverage    # Vitest with V8 coverage
npm run smoke            # Full release preflight
```

<Details>
<summary>📋 <strong>Was die Pipeline validiert</strong></summary>

- ✅ Kompetenzvalidierung und Metadatengenerierung
- ✅ Kritische Sicherheitskontrolle bei der einheimischen Aufnahme vor der PR-Zusammenführung
- ✅ Taxonomie-Normalisierungs- und Rekategorisierungstools
- ✅ Katalog- und Archiverstellung
- ✅ Legacy-Integrationstests (Python PTY + Node TUI-Behauptungen)
- ✅ Vitest-Einheitensuite für Katalogkernbewertung, Such- und Filterlogik
- ✅ API-, MCP- und A2A-Bootpfade
- ✅ Archivüberprüfung
- ✅ OpenAPI 3.1-Vertrag, der über die Swagger-Benutzeroberfläche unter „/docs“ bereitgestellt wird
- ✅ Paket-Preflight mit „npm pack --dry-run“.</details>

<Details>
<summary>🔐 <strong>Loshaltung</strong></summary>

| Kontrolle | Beschreibung |
|:--------|:-----------|
| 🔒 SHA-256-Prüfsummen | Prüfsummenmanifeste für alle Archive |
| ✍️ Signierte Artefakte | Abgelöste Signaturen auf Release-Artefakten |
| 🤖 CI-erzwungen | Release-Verifizierung im CI vor Veröffentlichung |
| 🦠 Scannertore | ClamAV und VirusTotal sind beide bei der Veröffentlichungsüberprüfung vor der Veröffentlichung erforderlich |
| 📦 GitHub-Veröffentlichung | Automatisierte GitHub-Release-Generierung |
| 📋 NPM-Veröffentlichung | Nur aus verifiziertem Tarball |
| 🔄 Automatische Freigabe | Bei Qualifikationsfertigkeiten verschmilzt mit „Haupt“ |

**Die automatische Freigabe wird nur ausgelöst, wenn sich eine Zusammenführung ändert:**
- `skills/*/**`
- `skills_omni/*/**`
- „data/bundles.json“.

Nur-Dokument-Änderungen lösen**keine**Paketveröffentlichung aus.</details>

<Details>
<summary>🛡️ <strong>Endlagersicherheitsvertrag</strong></summary>

- Bei der nativen Aufnahme werden keine Fähigkeiten mit kritischen Sicherheitsbefunden akzeptiert.
– Der statische Validator blockiert Muster mit hohem Risiko, bevor ein Skill in den öffentlichen Katalog aufgenommen werden kann.
- Zu den aktuellen Hardblockern gehören:
  - Remote-Inhalte werden direkt an die Shell-Ausführung weitergeleitet
  – Anweisungen, die versuchen, Eingabeaufforderungen, Geheimnisse oder verborgenen Laufzeitkontext preiszugeben
– Veröffentlichte Veröffentlichungen durchlaufen eine strengere zweite Ebene:
  - ClamAV-Scannen
  - VirusTotal-Hash-Suche
  - Überprüfung der Archivprüfsumme
  - Separate Signaturüberprüfung im Release-CI

Dies ist ein Vorteil für den Benutzer, nicht nur für Gebäudeinstallationen: Der Katalog bevorzugt installierbare Fertigkeiten, die bereits die gleiche restriktive Sicherheitsschleuse passiert haben, die das Projekt intern verwendet.</details>

## Beitragen

Awesome Omni Skills akzeptiert die Aufnahme nativer Upstream-Skills unter „skills/“.

| Regel | Einzelheiten |
|:-----|:--------|
| 📥 Native Aufnahme | Kann grob sein, in jeder Sprache verfasst sein und von direkten PRs oder überprüften externen Importzweigen stammen |
| ✨ Kuratierte Ausgabe | `skills_omni/` reserviert für durch Automatisierung erstellte Omni-Derivate |
| 🌍 Upstream-Repo-Vorschläge | Fügen Sie eine Zeile in [REPOSITORY-SOURCES.md](REPOSITORY-SOURCES.md) hinzu oder aktualisieren Sie sie, um ein Repository für die Überprüfung durch den Betreiber zu nominieren. Repo-First-Vorschläge können jetzt „branch=auto“ und „skills_path=auto“ | verwenden
| 🚫 Manuelle Änderungen | Öffentliche manuelle Änderungen an „skills_omni/“ werden abgelehnt |
| 🔄 Wiederaufbereitung | Der private Enhancer verarbeitet native Änderungen erneut und aktualisiert die kuratierte Baseline |

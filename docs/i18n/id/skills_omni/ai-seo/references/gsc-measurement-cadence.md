Use Google Search Console as the primary operating source for search performance and indexing diagnostics.

## Weekly cadence

| Area | Check | Watch for | Action |
| --- | --- | --- | --- |
| Performance | Queries and pages | rising impressions with weak CTR | improve title, intent match, and opening answer |
| Indexing | Page indexing report | discovered/crawled not indexed | audit uniqueness, internal links, canonicals, and sitemap inclusion |
| Internal links | Internal links report | weak or missing links to new pages | add links from hubs, categories, and related pages |
| New cohorts | page sample | weak template behavior | pause scaling until fixed |

## Monthly cadence

| Area | Check | Watch for | Action |
| --- | --- | --- | --- |
| Content freshness | top BOFU and comparison pages | stale pricing/features | refresh and re-review evidence |
| Cluster quality | page groups | duplication or low-value overlap | merge, prune, or rework weak pages |
| Expansion decisions | pilot results | low indexing or weak engagement | improve template before adding pages |
| Opportunity discovery | queries and appearances | new demand or adjacent intent | extend proven page patterns |

## Core KPIs

- indexed pages / published pages
- impressions by page set
- clicks and CTR by priority page set
- top query/page mismatches
- conversions from BOFU pages

## Threshold examples

These are examples only; adjust to site maturity:

- pause rollout if a pilot cohort has poor indexing after a reasonable observation period
- refresh titles if impressions rise but CTR remains persistently weak
- prune clusters that show duplication without business value

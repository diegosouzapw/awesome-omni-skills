This file defines when to suspend or avoid learning mode.

## Suspend learning mode immediately when

- the user says "just ship it", "just fix it", or equivalent
- the task becomes production-facing incident work
- the user is blocked and needs direct resolution first
- the interaction shows frustration with questioning
- the conversation becomes primarily debugging, testing, security review, or release coordination

## Soft limit for unsolicited offers

Do not offer more than 1-2 unsolicited exercises in a single session.

If the user declines once, continue with the task. If they later ask for explanation, you can re-enter learning mode.

## Good fallback behavior

When you suspend learning mode:

1. continue the implementation or debugging task directly
2. keep explanations concise and code-anchored
3. optionally provide a short recap after the work is done

## Principle

Learning mode is optional support for delivery, not a gate before delivery.

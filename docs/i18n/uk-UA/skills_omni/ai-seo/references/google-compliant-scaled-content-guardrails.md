This skill supports AI-assisted publishing only when the output is useful, original enough, and created for users first.

## Acceptable direction

Good scaled publishing usually has:

- a clear user task the page helps complete
- unique value per page
- original synthesis, data, comparisons, or utility
- visible facts that can be checked
- editorial review before scale

## High-risk patterns

Avoid:

- mass-generated near-duplicate pages
- doorway-like pages targeting slight query variants with little added value
- fake reviews or invented migration claims
- unsupported superiority language
- copying competitor review summaries without attribution or verification
- publishing large sets before sampling quality

## Operator gates before scale

Confirm all of the following:

- Why does this page deserve to exist separately?
- What visible element is unique on each page?
- What claim types require verification?
- Who reviews factual accuracy?
- What pages will be tested first before scaling?

## Rule of thumb

AI assistance is not the problem by itself. Low-value scaled content is the problem.

Use this note to route work after Figma MCP context has been gathered.

## Stay in `@figma` when

- setup, auth, or file access is still failing
- the exact node is still unclear
- screenshots or assets still need validation
- the implementation brief is incomplete

## Route to `@figma-implement-design` when

- the exact target has been confirmed
- structure and screenshot have been gathered
- required assets are identified or fetched
- the next step is production UI implementation

## Route to accessibility work when

- the UI is understood but semantic structure, focus behavior, contrast, or state communication need review

## Route to frontend or framework skills when

- the remaining work is component implementation, styling integration, state wiring, routing, or framework-specific code

## Route to testing or QA skills when

- implementation is complete and the next step is verification, regression checking, or interaction testing

## Handoff minimum packet

Always include:

- exact file URL
- exact node URL or node ID
- screenshot confirmation
- assets fetched or still missing
- unresolved ambiguities

## Kom hurtigt i gang

> 🔍**Søgte efter AI-kodningsfærdigheder, Claude Code-færdigheder, Cursorfærdigheder, Codex CLI-færdigheder, Gemini CLI-færdigheder, Antigravity-færdigheder eller installerbare 'SKILL.md'-biblioteker?**
>**Du er det rigtige sted.**👇### 1️⃣ What is this?

Awesome Omni Skills er et**installerbart færdighedskatalog og runtime**for AI-kodningsassistenter. I sin kerne er det et offentligt lager af genbrugelige `SKILL.md`-spilbøger, et kurateret lager af forbedrede bedste praksis-derivater og CLI-, API-, MCP- og A2A-runtime-overflader, der distribuerer og driver det samme katalog.

Native indtag kommer ind i `færdigheder/` på to måder: direkte bidragyder-PR'er og reviderede lagerbaserede ekstern-import-PR'er. Kurerede derivater, der kun er på engelsk, foreslås derefter separat i `skills_omni/` af den private forbedringspipeline.

## 🧩 Core Concepts

Før du sammenligner bundter eller vælger en installationssti, hjælper det at forstå disse fem byggeklodser:

| Koncept | Hvad det betyder |
|:--------|:------------|
| 🧠**færdigheder**| Genanvendelige `SKILL.md`-spillebøger, der lærer en assistent, hvordan man udfører et workflow godt |
| 📦**Katalogartefakter**| Genererede JSON- og arkivoutput, der muliggør søgning, sammenligning, download og installation |
| 🔌**MCP Config**| Konfiguration på klientsiden for assistenter til at opdage Awesome Omni Skills gennem MCP-værktøjer |
| 🤖**A2A Runtime**| Agent-til-agent orkestrering til opdagelse, anbefaling og overdragelse af installationsplan |
| ✨**Kurateret output**| `skills_omni/` — den Awesome Omni Skills-vedligeholdte forbedrede overflade, adskilt fra det oprindelige opstrømsindtag |

>**📝 Indfødt/kurateret politik:**
> - `skills/` accepterer oprindelige opstrømsindtag på ethvert sprog, uanset om det er indsendt direkte eller foreslået af den private ekstern synkroniserings-runtime
> - `skills_omni/` er altid kurateret og udgivet på engelsk
> - `skills_omni/` er en envejsoverflade og går ikke tilbage i native indtag
> - direkte bidragydere PR'er og lagerbaseret eksternt indtag deler den samme validator, metadataopdatering og forstærkersti---

## 💡 Why Awesome Omni Skills

>**Ikke kun "endnu et lager med færdigheder i mapper."**
> Awesome Omni Skills har en stærkere kontrakt og en bredere runtime-overflade.

| Hvis du vil… | 📁 Typisk færdighedsrepo | ✨ Fantastiske Omni-færdigheder |
|:-------------|:---------------------|:--------|
| Installer i en rigtig assistent | Manuel kopi eller brugerdefineret script | `npx awesome-omni-skills`, guidet installation, visuel UI, selektiv `--skill` og `--bundle` |
| Søg og sammenlign færdigheder | Gennemse markdown manuelt | Genereret katalog, filtrering, bundtplanlægning, søgning, sammenligning og anbefaling |
| Brug de samme data på tværs af værktøjer | Separat logik pr. værktøj | Delte manifester og katalog for CLI, API, MCP og A2A |
| Konfigurer MCP-klienter | Håndrediger filer | `config-mcp`, lokale sidevognsforhåndsvisninger, genererede opskrifter og tilladelseslistede skrivninger |
| Trust repository indhold | Minimal eller ad hoc fnug | Kritisk statisk sikkerhedsport blokerer usikre mønstre såsom fjernindhold, der sendes ind i shell eller instruktioner, der forsøger at afsløre prompter, hemmeligheder eller skjult kontekst |
| Tillidsfrigivelser | Bedste emballage | Kontrolsummer, underskrevne arkiver, ClamAV + VirusTotal verificeret udgivelses-CI og udgiv forhåndskontrol |
| Kurater fællesskabsindtag | Uanset hvad der lander forbliver som det er | Native indtag i `skills/`, kuraterede engelske derivater i `skills_omni/` med tilskrivning |---

## 🛡️ Security and Release Posture

> Awesome Omni Skills sender en stærkere udgivelses- og verifikationshistorie end et almindeligt nedskrivningsarkiv.### 🧪 Validation and Smoke Checks

```bash
npm run validate         # Skill validation and metadata generation
npm run build            # Full build pipeline
npm test                 # Legacy integration tests + Vitest unit suite
npm run test:unit        # Fast Vitest unit tests only (~500ms)
npm run test:coverage    # Vitest with V8 coverage
npm run smoke            # Full release preflight
```

<detaljer>
<summary>📋 <strong>Hvad pipelinen validerer</strong></summary>

- ✅ Færdighedsvalidering og metadatagenerering
- ✅ Kritisk sikkerhedsport på native indtag før PR-fusion
- ✅ Taksonomi normalisering og rekategoriseringsværktøj
- ✅ Katalog og arkivgenerering
- ✅ Ældre integrationstests (Python PTY + Node TUI-påstande)
- ✅ Vitest-enhedssuite til katalogkernescoring, søgning og filtreringslogik
- ✅ API-, MCP- og A2A-startstier
- ✅ Arkivverifikation
- ✅ OpenAPI 3.1-kontrakt serveret via Swagger UI på `/docs`
- ✅ Pakke preflight med `npm pack --dry-run`</details>

<detaljer>
<resumé>🔐 <strong>Slip stilling</strong></summary>

| Kontrol | Beskrivelse |
|:--------|:--------|
| 🔒 SHA-256 kontrolsummer | Kontrolsummanifester for alle arkiver |
| ✍️ Signerede artefakter | Fritliggende signaturer på udgivelsesartefakter |
| 🤖 CI-håndhævet | Frigivelsesbekræftelse i CI før offentliggørelse |
| 🦠 Scannerporte | ClamAV og VirusTotal kræves begge i udgivelsesbekræftelse før publicering |
| 📦 GitHub-udgivelse | Automatiseret GitHub-udgivelsesgenerering |
| 📋 npm udgivelse | Kun fra verificeret tarball |
| 🔄 Autofrigivelse | Ved kvalificerende fusionerer færdighed til `main` |

**Automatisk udgivelse udløses kun, når en fletning ændres:**
- `færdigheder/*/**`
- `skills_omni/*/**`
- `data/bundles.json`

Ændringer kun til dokumenter**udløser ikke**pakkeudgivelse.</details>

<detaljer>
<resumé>🛡️ <strong>Depotsikkerhedskontrakt</strong></summary>

- Native intake accepterer ikke færdigheder med kritiske sikkerhedsfund.
- Den statiske validator blokerer højrisikomønstre, før en færdighed kan smelte sammen i det offentlige katalog.
- De nuværende hårde blokere inkluderer:
  - Fjernindhold, der sendes direkte til shell-udførelse
  - instruktioner, der forsøger at afsløre prompter, hemmeligheder eller skjult runtime-kontekst
- Udgivne udgivelser gennemgår et strengere andet lag:
  - ClamAV-scanning
  - VirusTotal hash-opslag
  - arkiv kontrolsum verifikation
  - adskilt signaturverifikation i udgivelses-CI

Dette er en brugervendt fordel, ikke kun bygge VVS: Kataloget favoriserer installerbare færdigheder, der allerede har bestået den samme restriktive sikkerhedsport, som projektet bruger internt.</details>

## 🗂️ Repository Layout

| Sti | Formål |
|:-----|:--------|
| 📂 `færdigheder/` | Kanonisk forfattede færdigheder og indbygget indtag fra direkte PR'er eller revideret lagerbaseret import |
| ✨ `skills_omni/` | Kurerede Omni-vedligeholdte forbedrede derivater |
| 📖 `dokumenter/` | Bruger-, bidragyder-, arkitektur-, operations- og specifikationsdokumentation |
| 📦 `dist/` | Genererede manifester, bundter, kataloger og arkiver |
| 📁 `data/` | Bundledefinitioner og statiske understøttende data |
| 🧠 `pakker/katalog-kerne/` | Delt katalogkørsel med `ICatalogStorageAdapter` afhængighedsinjektion |
| 🌐 `pakker/server-api/` | Skrivebeskyttet HTTP API med OpenAPI/Swagger UI på `/docs` |
| 🔌 `pakker/server-mcp/` | MCP-server og lokal sidevogn |
| 🤖 `pakker/server-a2a/` | A2A runtime og opgave orkestrering |
| 🖥️ `pakker/cli/` | Unified CLI entrypoints, installer logik og Ink visual TUI (ESM-native) |
| ⚙️ `værktøjer/scripts/` | Validering, generering, frigivelse og testscripts |
| 🧪 `vitest.workspace.js` | Vitest monorepo arbejdsområde konfiguration |

>**ℹ️**`dist/` er bevidst versioneret, fordi de genererede artefakter er en del af installations-, API-, MCP-, A2A-, røg- og frigivelseskontrakten.---

## Bidrage

Awesome Omni Skills accepterer indbygget opstrøms færdighedsindtag under "færdigheder/".

| Regel | Detaljer |
|:-----|:--------|
| 📥 Native indtag | Kan være grov, skrevet på ethvert sprog og kan komme fra direkte PR'er eller reviderede eksterne importgrene |
| ✨ Kurateret output | `skills_omni/` forbeholdt automatiserings-forfattede Omni-derivater |
| 🌍 Opstrøms repo-forslag | Tilføj eller opdater én række i [REPOSITORY-SOURCES.md](REPOSITORY-SOURCES.md) for at nominere et lager til operatørgennemgang; repo-first forslag kan nu bruge `branch=auto` og `skills_path=auto` |
| 🚫 Manuelle redigeringer | Offentlige manuelle redigeringer af `skills_omni/` afvises |
| 🔄 Genbehandling | Privat forstærker genbehandler native ændringer og opdaterer kureret baseline |

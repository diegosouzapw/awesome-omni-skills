## Snel starten

> 🔍**Gezocht naar AI-codeervaardigheden, Claude Code-vaardigheden, Cursor-vaardigheden, Codex CLI-vaardigheden, Gemini CLI-vaardigheden, Antigravity-vaardigheden of installeerbare `SKILL.md`-bibliotheken?**
>**Je bent op de juiste plaats.**👇### 1️⃣ What is this?

Awesome Omni Skills is een**installeerbare vaardighedencatalogus en runtime**voor AI-codeerassistenten. In de kern is het een openbare opslagplaats van herbruikbare 'SKILL.md'-playbooks, een samengestelde opslagplaats van verbeterde best-practice-derivaten, en de CLI-, API-, MCP- en A2A-runtime-oppervlakken die dezelfde catalogus distribueren en beheren.

Native intake omvat 'vaardigheden/' op twee manieren: PR's van directe bijdragers en beoordeelde op repository's gebaseerde externe import-PR's. Gecureerde derivaten die alleen in het Engels beschikbaar zijn, worden vervolgens afzonderlijk voorgesteld in `skills_omni/` door de particuliere verbeteringspijplijn.

<details>
<summary>📋 <strong>Wat is inbegrepen</strong></summary>

| Onderdeel | Beschrijving |
|:----------|:-----------|
| 🧠**Vaardigheden**| Herbruikbare op 'SKILL.md' gebaseerde draaiboeken voor AI-assistenten, inclusief native intake van PR's van directe bijdragers en beoordeelde op repository's gebaseerde externe import-PR's |
| 📦**Manifesten**| Gegenereerde JSON-manifesten, bundels en archieven |
| 🧭**Begeleide installatie**| Interactieve TTY en visuele terminalinstallatiestromen |
| 🌐**Catalogus-API**| Alleen-lezen HTTP-API voor zoeken, ontdekken en downloaden |
| 🔌**MCP-server**| Ontdekkings-, aanbevelings- en klantbewuste configuratietools |
| 🤖**A2A Runtime**| Taakorkestratie van agent tot agent |
| ✨**Verbeteringspijplijn**| Private Enhancer publiceert samengestelde Engelse derivaten in `skills_omni/` |</details>

## 🧩 Core Concepts

Voordat u bundels vergelijkt of een installatiepad kiest, helpt het begrijpen van deze vijf bouwstenen:

| Concept | Wat het betekent |
|:--------|:-------------|
| 🧠**Vaardigheden**| Herbruikbare `SKILL.md`-playbooks die een assistent leren hoe hij een workflow goed kan uitvoeren |
| 📦**Catalogusartefacten**| Gegenereerde JSON- en archiefuitvoer die zoeken, vergelijken, downloaden en installeren mogelijk maken |
| 🔌**MCP-configuratie**| Configuratie aan de clientzijde voor assistenten om Awesome Omni Skills te ontdekken via MCP-tools |
| 🤖**A2A Runtime**| Agent-naar-agent-orkestratie voor detectie, aanbeveling en overdracht van installatieplannen |
| ✨**Samengestelde uitvoer**| `skills_omni/` — het door Awesome Omni Skills onderhouden verbeterde oppervlak, los van de native upstream-inname |

>**📝 Native/Curated-beleid:**
> - `skills/` accepteert native upstream-inname in elke taal, ongeacht of deze rechtstreeks wordt ingediend of wordt voorgesteld door de particuliere externe synchronisatieruntime
> - `skills_omni/` wordt altijd samengesteld en gepubliceerd in het Engels
> - `skills_omni/` is eenrichtingsverkeer en loopt niet terug naar de oorspronkelijke inname
> - PR's van directe bijdragers en op repository's gebaseerde externe inname delen dezelfde validator, vernieuwing van metagegevens en verbeteringspad---

| Inhoud van trustrepository | Minimale of ad hoc pluisvorming | Kritieke statische beveiligingspoort blokkeert onveilige patronen, zoals externe inhoud die in de shell wordt doorgesluisd of instructies die aanwijzingen, geheimen of verborgen context proberen bloot te leggen |
| Trust-releases | Best-effort-verpakking | Checksums, ondertekende archieven, ClamAV + VirusTotal geverifieerde release CI en preflight publiceren |
| Curate gemeenschapsinname | Welk land dan ook blijft zoals het is | Native intake in `skills/`, samengestelde Engelse afgeleiden in `skills_omni/` met toeschrijving |---

## 🛡️ Security and Release Posture

> Awesome Omni Skills levert een sterker release- en verificatieverhaal dan een eenvoudige opslagplaats voor prijsverlagingen.### 🧪 Validation and Smoke Checks

```bash
npm run validate         # Skill validation and metadata generation
npm run build            # Full build pipeline
npm test                 # Legacy integration tests + Vitest unit suite
npm run test:unit        # Fast Vitest unit tests only (~500ms)
npm run test:coverage    # Vitest with V8 coverage
npm run smoke            # Full release preflight
```

<details>
<summary>📋 <strong>Wat de pijplijn valideert</strong></summary>

- ✅ Vaardigheidsvalidatie en het genereren van metadata
- ✅ Kritieke beveiliging bij native intake vóór PR-fusie
- ✅ Tooling voor normalisatie en hercategorisatie van taxonomie
- ✅ Catalogus- en archiefgeneratie
- ✅ Legacy-integratietests (Python PTY + Node TUI-beweringen)
- ✅ Vitest-unitsuite voor cataloguskernscore-, zoek- en filterlogica
- ✅ API-, MCP- en A2A-opstartpaden
- ✅ Archiefverificatie
- ✅ OpenAPI 3.1-contract geserveerd via Swagger UI op `/docs`
- ✅ Pakketpreflight met `npm pack --dry-run`</details>

<details>
<summary>🔐 <strong>Laathouding los</strong></summary>

| Controle | Beschrijving |
|:--------|:-----------|
| 🔒 SHA-256-controlesommen | Checksum-manifesten voor alle archieven |
| ✍️ Gesigneerde artefacten | Vrijstaande handtekeningen op release-artefacten |
| 🤖 CI-afgedwongen | Vrijgaveverificatie in CI vóór publicatie |
| 🦠 Scannerpoorten | ClamAV en VirusTotal zijn beide vereist voor releaseverificatie vóór publicatie |
| 📦 GitHub-release | Geautomatiseerde generatie van GitHub-releases |
| 📋 npm publicatie | Alleen van geverifieerde tarball |
| 🔄 Automatische vrijgave | Bij kwalificatie worden vaardigheden samengevoegd tot `main` |

**Automatische vrijgave wordt alleen geactiveerd wanneer een samenvoeging verandert:**
- `vaardigheden/*/**`
- `vaardigheden_omni/*/**`
- `data/bundels.json`

Wijzigingen in alleen documenten leiden**niet**tot pakketpublicatie.</details>

<details>
<summary>🛡️ <strong>Repository-veiligheidscontract</strong></summary>

- Native intake accepteert geen vaardigheden met kritische beveiligingsbevindingen.
- De statische validator blokkeert patronen met een hoog risico voordat een vaardigheid kan worden opgenomen in de openbare catalogus.
- De huidige harde blokkers omvatten:
  - inhoud op afstand die rechtstreeks naar de shell-uitvoering wordt geleid
  - instructies die aanwijzingen, geheimen of verborgen runtimecontext proberen te onthullen
- Gepubliceerde releases doorlopen een strengere tweede laag:
  - ClamAV-scannen
  - VirusTotal-hash opzoeken
  - archiefcontrolesomverificatie
  - vrijstaande handtekeningverificatie in release CI

Dit is een voordeel voor de gebruiker, niet alleen voor het bouwen van loodgieterswerk: de catalogus geeft de voorkeur aan installeerbare vaardigheden die al door dezelfde beperkende beveiligingspoort zijn gegaan die het project intern gebruikt.</details>

## Bijdragen

Awesome Omni Skills accepteert native upstream-vaardigheidsinname onder `skills/`.

| Regel | Details |
|:-----|:--------|
| 📥Native instroom | Kan ruw zijn, geschreven in elke taal, en kan afkomstig zijn van directe PR's of beoordeelde externe importfilialen |
| ✨ Samengestelde uitvoer | `skills_omni/` gereserveerd voor op automatisering geschreven Omni-derivaten |
| 🌍 Upstream repo-voorstellen | Voeg één rij toe of update deze in [REPOSITORY-SOURCES.md](REPOSITORY-SOURCES.md) om een ​​repository te nomineren voor beoordeling door de operator; repo-first voorstellen kunnen nu `branch=auto` en `skills_path=auto` | gebruiken
| 🚫 Handmatige bewerkingen | Openbare handmatige bewerkingen van `skills_omni/` worden afgewezen |
| 🔄Herverwerking | Private Enhancer verwerkt oorspronkelijke wijzigingen opnieuw en vernieuwt de samengestelde basislijn |

> 📖**Begin met:**[Handleiding voor bijdragen](CONTRIBUTING.md) · [Skill PR Workflow](docs/contributors/SKILL-PR-WORKFLOW.md) · [Repositorybronnen](REPOSITORY-SOURCES.md)---

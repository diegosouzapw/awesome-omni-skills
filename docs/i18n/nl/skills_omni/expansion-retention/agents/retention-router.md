Use this note when the request starts in expansion-retention but should move elsewhere.

## Stay in this skill when

- the task is mainly retention diagnosis
- the task is mainly expansion trigger design
- the task is mainly renewal operating design
- the output is a strategy, playbook, checklist, or owner map

## Hand off when

### Pricing or packaging dominates
Route to a pricing skill if the user needs:
- packaging redesign
- discount policy
- monetization architecture
- pricing experiments

### Analytics implementation dominates
Route to an analytics or instrumentation skill if the user needs:
- event taxonomy design
- dashboard build-out
- warehouse SQL implementation
- retention model instrumentation

### CRM or CS tooling implementation dominates
Route to a RevOps or platform skill if the user needs:
- workflow automation
- object/schema changes
- scoring implementation in a platform
- routing or notification setup

### Legal or compliance interpretation dominates
Route to legal/compliance review if the user needs:
- consent interpretation
- privacy-law interpretation
- contract-language advice
- collections or notice requirements

## Handoff rule

Preserve:
- the current diagnosis
- assumptions
- segment definitions
- unresolved risks
- requested implementation outcome

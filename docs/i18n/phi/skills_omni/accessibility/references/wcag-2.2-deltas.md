This skill targets WCAG 2.1 as the baseline. Use this note to flag nearby improvements without silently changing the required scope.

## Useful WCAG 2.2 checks to notice

- **Focus Appearance**: focus indicator is technically present but too subtle to use reliably.
- **Dragging Movements**: drag-only interactions lack an alternative method.
- **Target Size (Minimum)**: small hit areas create avoidable pointer difficulty.
- **Accessible Authentication** concerns where cognitive burden is unnecessarily high.

## How to report

Use wording like:

> Not a WCAG 2.1 failure as scoped here, but this is a relevant WCAG 2.2 improvement opportunity.

Do not present WCAG 2.2 as mandatory unless the product requirement says so.

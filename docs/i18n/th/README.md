## เริ่มต้นอย่างรวดเร็ว

> 🔍**ค้นหาทักษะการเข้ารหัส AI, ทักษะ Claude Code, ทักษะเคอร์เซอร์, ทักษะ Codex CLI, ทักษะ Gemini CLI, ทักษะต้านแรงโน้มถ่วง หรือไลบรารี `SKILL.md` ที่ติดตั้งได้?**
>**คุณมาถูกที่แล้ว**👇### 1️⃣ What is this?

Awesome Omni Skills คือ**แค็ตตาล็อกทักษะและรันไทม์ที่ติดตั้งได้**สำหรับผู้ช่วยเขียนโค้ด AI หัวใจหลักของมันคือพื้นที่เก็บข้อมูลสาธารณะของ playbooks `SKILL.md` ที่นำมาใช้ซ้ำได้ ซึ่งเป็นพื้นที่เก็บข้อมูลที่รวบรวมไว้สำหรับอนุพันธ์แนวปฏิบัติที่ดีที่สุดที่ได้รับการปรับปรุง และพื้นผิวรันไทม์ CLI, API, MCP และ A2A ที่แจกจ่ายและดำเนินการแค็ตตาล็อกเดียวกัน

การรับข้อมูลโดยธรรมชาติเข้าสู่ `ทักษะ/` ในสองวิธี: PR ผู้สนับสนุนโดยตรง และ PR นำเข้าภายนอกตามพื้นที่เก็บข้อมูลที่ตรวจสอบแล้ว จากนั้นจะมีการเสนออนุพันธ์เฉพาะภาษาอังกฤษที่ได้รับการดูแลจัดการแยกเป็น `skills_omni/` โดยไปป์ไลน์การเพิ่มประสิทธิภาพส่วนตัว

## 🧩 Core Concepts

ก่อนที่จะเปรียบเทียบบันเดิลหรือเลือกเส้นทางการติดตั้ง การทำความเข้าใจบล็อคส่วนประกอบทั้งห้านี้จะช่วยได้:

| แนวคิด | ความหมาย |
|:--------|:-------------|
| 🧠**ทักษะ**| Playbooks `SKILL.md` ที่นำกลับมาใช้ใหม่ได้ซึ่งสอนผู้ช่วยถึงวิธีดำเนินการเวิร์กโฟลว์ให้ดี |
| 📦**สิ่งประดิษฐ์แคตตาล็อก**| JSON ที่สร้างขึ้นและเอาต์พุตการเก็บถาวรทำให้สามารถค้นหา เปรียบเทียบ ดาวน์โหลด และติดตั้ง |
| 🔌**การกำหนดค่า MCP**| การกำหนดค่าฝั่งไคลเอ็นต์เพื่อให้ผู้ช่วยค้นพบทักษะ Omni ที่ยอดเยี่ยมผ่านเครื่องมือ MCP
| 🤖**รันไทม์ A2A**| การประสานงานระหว่างตัวแทนถึงตัวแทนสำหรับการค้นพบ คำแนะนำ และการส่งมอบแผนการติดตั้ง |
| ✨**ผลลัพธ์ที่คัดสรรแล้ว**| `skills_omni/` — พื้นผิวที่ได้รับการปรับปรุงซึ่งรักษาทักษะ Omni ที่ยอดเยี่ยม แยกจากการรับต้นทางดั้งเดิม |

>**📝 นโยบายเนทีฟ/คัดสรร:**
> - `ทักษะ/` ยอมรับการรับอัปสตรีมดั้งเดิมในภาษาใดๆ ไม่ว่าจะส่งโดยตรงหรือเสนอโดยรันไทม์การซิงค์ภายนอกส่วนตัว
> - `skills_omni/` ได้รับการดูแลจัดการและเผยแพร่เป็นภาษาอังกฤษเสมอ
> - `skills_omni/` เป็นพื้นผิวทางเดียว และไม่วนกลับเข้าสู่การบริโภคดั้งเดิม
> - PR ผู้สนับสนุนโดยตรงและการรับข้อมูลภายนอกตามพื้นที่เก็บข้อมูลแบ่งปันเครื่องมือตรวจสอบ การรีเฟรชข้อมูลเมตา และเส้นทางเครื่องมือเพิ่มประสิทธิภาพเดียวกัน---

| หากคุณต้องการ… | 📁 การซื้อคืนทักษะทั่วไป | ✨ ทักษะ Omni ที่ยอดเยี่ยม |

| ใช้ข้อมูลเดียวกันในเครื่องมือต่างๆ | แยกตรรกะต่อเครื่องมือ | รายการและแค็ตตาล็อกที่แชร์สำหรับ CLI, API, MCP และ A2A |
| กำหนดค่าไคลเอนต์ MCP | ไฟล์แก้ไขด้วยมือ | `config-mcp`, ตัวอย่างรถเทียมข้างรถจักรยานยนต์ในเครื่อง, สูตรอาหารที่สร้างขึ้น และการเขียนที่อนุญาต |
| เชื่อถือเนื้อหาที่เก็บข้อมูล | ขุยน้อยที่สุดหรือเฉพาะกิจ | ประตูรักษาความปลอดภัยแบบคงที่ที่สำคัญจะบล็อกรูปแบบที่ไม่ปลอดภัย เช่น เนื้อหาระยะไกลที่ส่งไปยังเชลล์หรือคำสั่งที่พยายามเปิดเผยข้อความแจ้ง ข้อมูลลับ หรือบริบทที่ซ่อนอยู่
| เผยแพร่ความน่าเชื่อถือ | บรรจุภัณฑ์ที่พยายามอย่างดีที่สุด | Checksums, ไฟล์เก็บถาวรที่ลงนาม, ClamAV + VirusTotal CI รุ่นที่ได้รับการตรวจสอบแล้ว และเผยแพร่ preflight |
| คัดสรรการบริโภคของชุมชน | ไม่ว่าแผ่นดินใดจะคงอยู่ดังที่เป็นอยู่ | การเรียนรู้ภาษาพื้นเมืองใน `ทักษะ/`, อนุพันธ์ภาษาอังกฤษที่คัดสรรมาใน `skills_omni/` พร้อมการระบุแหล่งที่มา |---

| 📋 สิ่งพิมพ์ NPM | จาก tarball ที่ตรวจสอบแล้วเท่านั้น |
| 🔄 ปล่อยอัตโนมัติ | เมื่อทักษะที่มีคุณสมบัติเหมาะสมจะรวมเข้ากับ `main` |

**ทริกเกอร์การเผยแพร่อัตโนมัติเฉพาะเมื่อการเปลี่ยนแปลงการรวม:**
- `ทักษะ/*/**`
- `ทักษะ_omni/*/**`
- `data/bundles.json`

การเปลี่ยนแปลงเฉพาะเอกสาร**ไม่**ทริกเกอร์การเผยแพร่แพ็กเกจ</details>

<รายละเอียด>
<summary>🛡️ <strong>สัญญาความปลอดภัยของพื้นที่เก็บข้อมูล</strong></summary>

- การรับเข้าเรียนโดยเจ้าของภาษาไม่ยอมรับทักษะที่มีผลการสืบค้นด้านความปลอดภัยที่สำคัญ
- เครื่องมือตรวจสอบแบบคงที่จะบล็อกรูปแบบที่มีความเสี่ยงสูงก่อนที่ทักษะจะสามารถรวมเข้ากับแคตตาล็อกสาธารณะได้
- ฮาร์ดบล็อกเกอร์ในปัจจุบันประกอบด้วย:
  - เนื้อหาระยะไกลส่งตรงไปยังการดำเนินการของเชลล์
  - คำสั่งที่พยายามเปิดเผยข้อความแจ้ง ข้อมูลลับ หรือบริบทรันไทม์ที่ซ่อนอยู่
- การเผยแพร่ที่เผยแพร่จะต้องผ่านเลเยอร์ที่สองที่เข้มงวดกว่า:
  - การสแกน ClamAV
  - ค้นหาแฮชของ VirusTotal
  - เก็บถาวรการตรวจสอบการตรวจสอบ
  - การตรวจสอบลายเซ็นที่แยกออกมาใน CI รุ่น

นี่เป็นประโยชน์ที่ผู้ใช้เผชิญ ไม่ใช่เพียงสร้างระบบประปา: แค็ตตาล็อกสนับสนุนทักษะที่สามารถติดตั้งได้ซึ่งผ่านประตูรักษาความปลอดภัยแบบเข้มงวดแบบเดียวกับที่โปรเจ็กต์ใช้ภายในแล้ว</details>

## 🗂️ Repository Layout

| เส้นทาง | วัตถุประสงค์ |
|:-----|:--------|
| 📂 `ทักษะ/` | ทักษะที่เขียนโดย Canonical และการรับข้อมูลจาก PR โดยตรงหรือการนำเข้าตามพื้นที่เก็บข้อมูลที่ได้รับการตรวจสอบ |
| ✨ `skills_omni/` | Curated Omni-maintained อนุพันธ์ที่ได้รับการปรับปรุง |
| 📖 `เอกสาร/` | ผู้ใช้ ผู้ร่วมให้ข้อมูล สถาปัตยกรรม การดำเนินการ และเอกสารประกอบข้อมูลจำเพาะ |
| 📦 `dist/` | รายการ บันเดิล แค็ตตาล็อก และไฟล์เก็บถาวรที่สร้างขึ้น |
| 📁 `ข้อมูล/` | คำจำกัดความของ Bundle และข้อมูลสนับสนุนแบบคงที่ |
| 🧠 `แพ็คเกจ/แค็ตตาล็อก-คอร์/` | รันไทม์แคตตาล็อกที่ใช้ร่วมกันด้วยการฉีดการพึ่งพา `ICatalogStorageAdapter` |
| 🌐 `แพ็คเกจ/เซิร์ฟเวอร์-api/` | HTTP API แบบอ่านอย่างเดียวพร้อม OpenAPI/Swagger UI บน `/docs` |
| 🔌 `แพ็คเกจ/เซิร์ฟเวอร์-mcp/` | เซิร์ฟเวอร์ MCP และรถเทียมข้างรถจักรยานยนต์ |
| 🤖 `แพ็คเกจ/เซิร์ฟเวอร์-a2a/` | รันไทม์ A2A และการจัดการงาน |
| 🖥️ `แพ็คเกจ/cli/` | จุดเข้าใช้งาน CLI แบบรวม ตรรกะการติดตั้ง และ Ink Visual TUI (ESM-native) |
| ⚙️ `เครื่องมือ/สคริปต์/` | การตรวจสอบความถูกต้อง การสร้าง การเผยแพร่ และการทดสอบสคริปต์ |
| 🧪 `vite.workspace.js` | เยี่ยมชมการกำหนดค่าพื้นที่ทำงาน monorepo |

>**ℹ️**`dist/` ได้รับการกำหนดเวอร์ชันโดยตั้งใจเนื่องจากอาร์ติแฟกต์ที่สร้างขึ้นเป็นส่วนหนึ่งของสัญญาการติดตั้ง, API, MCP, A2A, Smoke และ Release---

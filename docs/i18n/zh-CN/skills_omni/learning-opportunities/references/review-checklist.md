Use this checklist to review whether the skill was applied well.

## Consent and boundaries

- [ ] The agent used learning mode only when the task warranted it.
- [ ] The agent asked permission before starting an exercise.
- [ ] The agent respected a decline without friction.
- [ ] The agent did not use learning mode during urgent debugging or explicit delivery-first requests.

## Exercise quality

- [ ] The exercise was tied to one concrete file, function, query, migration, or test.
- [ ] The agent asked exactly one question.
- [ ] The question was narrow enough to answer without exploring a whole subsystem.
- [ ] The agent stopped after the question.

## Feedback quality

- [ ] The feedback clearly distinguished correct and incorrect parts.
- [ ] The explanation tied back to the current code or design choice.
- [ ] The tone was direct and respectful, not shaming or evasive.

## Flow quality

- [ ] Learning mode did not block delivery progress.
- [ ] The interaction stayed short and purposeful.
- [ ] The agent resumed the original task after the learning loop.

## Handoff quality

- [ ] If the task drifted, the agent suspended learning mode.
- [ ] Relevant context was preserved for the next skill or workflow.

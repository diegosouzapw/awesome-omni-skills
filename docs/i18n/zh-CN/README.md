## 快速开始

> 🔍**搜索了 AI 编码技能、Claude Code 技能、光标技能、Codex CLI 技能、Gemini CLI 技能、反重力技能或可安装的“SKILL.md”库？**
>**您来对地方了。**👇### 1️⃣ What is this?

Awesome Omni Skills 是一个适用于 AI 编码助手的**可安装技能目录和运行时**。从本质上讲，它是可重用“SKILL.md”剧本的公共存储库、改进的最佳实践衍生品的精选存储库，以及分发和操作同一目录的 CLI、API、MCP 和 A2A 运行时界面。

本地摄入以两种方式进入“技能/”：直接贡献者 PR 和经过审查的基于存储库的外部导入 PR。然后，通过私人增强管道将精选的纯英语衍生品单独提议到“skills_omni/”中。

## 🧩 Core Concepts

在比较捆绑包或选择安装路径之前，了解这五个构建块有助于：

|概念 |这意味着什么 |
|:--------|:-------------|
| 🧠**技能**|可重复使用的“SKILL.md”手册，教助理如何很好地执行工作流程 |
| 📦**目录文物**|生成的 JSON 和存档输出支持搜索、比较、下载和安装 |
| 🔌**MCP 配置**|助手的客户端配置可通过 MCP 工具发现 Awesome Omni Skills |
| 🤖**A2A 运行时**|用于发现、推荐和安装计划切换的代理到代理编排 |
| ✨**策划输出**| `skills_omni/` — Awesome Omni Skills 维护的增强界面，独立于本地上游入口 |

>**📝 原生/策划政策：**
> - `skills/` 接受任何语言的本机上游接收，无论是直接提交还是由私有外部同步运行时建议
> - `skills_omni/` 始终以英语策划和发布
> - `skills_omni/` 是一个单向表面，不会循环回本机摄入
> - 直接贡献者 PR 和基于存储库的外部摄入共享相同的验证器、元数据刷新和增强器路径---

<详情>
<summary>🛡️ <strong>存储库安全合约</strong></summary>

- 本地招生不接受具有关键安全发现的技能。
- 静态验证器会在技能合并到公共目录之前阻止高风险模式。
- 当前的硬阻止程序包括：
  - 远程内容直接通过管道传输到 shell 执行中
  - 试图揭示提示、秘密或隐藏的运行时上下文的指令
- 发布的版本要经过更严格的第二层：
  - ClamAV扫描
  - VirusTotal 哈希查找
  - 存档校验和验证
  - 发布 CI 中分离的签名验证

这是面向用户的好处，而不仅仅是构建管道：该目录有利于已通过项目内部使用的相同限制性安全门的可安装技能。</details>

## 🗂️ Repository Layout

|路径|目的|
|:-----|:--------|
| 📂 `技能/` |规范的创作技能和直接 PR 或经过审查的基于存储库的导入的本地吸收 |
| ✨ `skills_omni/` |精选 Omni 维护的增强型衍生品 |
| 📖 `文档/` |用户、贡献者、架构、操作和规范文档 |
| 📦 `dist/` |生成的清单、捆绑包、目录和档案 |
| 📁 `数据/` |捆绑包定义和静态支持数据 |
| 🧠 `packages/catalog-core/` |具有“ICatalogStorageAdapter”依赖项注入的共享目录运行时 |
| 🌐 `packages/server-api/` | `/docs` 上具有 OpenAPI/Swagger UI 的只读 HTTP API |
| 🔌 `packages/server-mcp/` | MCP 服务器和本地 sidecar |
| 🤖 `packages/server-a2a/` | A2A 运行时和任务编排 |
| 🖥️ `packages/cli/` |统一 CLI 入口点、安装逻辑和 Ink 可视 TUI（ESM 原生）|
| ⚙️ `工具/脚本/` |验证、生成、发布和测试脚本 |
| 🧪 `vitest.workspace.js` | Vitest monorepo 工作区配置 |

>**ℹ️**`dist/` 是有意进行版本控制的，因为生成的工件是安装、API、MCP、A2A、smoke 和发布合约的一部分。---

## 贡献

Awesome Omni Skills 接受“技能/”下的本地上游技能吸收。

|规则|详情 |
|:-----|:--------|
| 📥 本土摄入 |可能很粗糙，用任何语言编写，并且可以来自直接 PR 或经过审查的外部导入分支 |
| ✨ 策划输出 | `skills_omni/` 保留给自动化创作的 Omni 衍生品 |
| 🌍 上游回购提案 |在 [REPOSITORY-SOURCES.md](REPOSITORY-SOURCES.md) 中添加或更新一行以指定存储库供操作员审核； repo-first 提案现在可以使用 `branch=auto` 和 `skills_path=auto` |
| 🚫 手动编辑 |对 `skills_omni/` 的公共手动编辑被拒绝 |
| 🔄 再处理 |私有增强器重新处理本机更改并刷新策划的基线 |

```text
Use @coding-guidelines to refactor <path> for <goal> without changing behavior. Inspect nearby code and tests first, preserve local conventions, avoid speculative abstractions, keep the diff tightly scoped, and run before/after validation for the touched area. Report unrelated issues separately instead of fixing them.
```

## Snabbstart

> 🔍**Sökade efter färdigheter i AI-kodning, Claude Code-färdigheter, Cursor-färdigheter, Codex CLI-färdigheter, Gemini CLI-färdigheter, Antigravity-färdigheter eller installerbara `SKILL.md`-bibliotek?**
>**Du har kommit rätt.**👇### 1️⃣ What is this?

Awesome Omni Skills är en**installerbar färdighetskatalog och körtid**för AI-kodningsassistenter. I grunden är det ett offentligt arkiv med återanvändbara "SKILL.md"-spelböcker, ett kurerat arkiv med förbättrade bästa praxisderivat och CLI, API, MCP och A2A runtime-ytor som distribuerar och driver samma katalog.

Native intake kommer in i `skills/` på två sätt: direkta bidragsgivare och granskade repository-baserade extern-import-PR. Kurerade endast engelska derivator föreslås sedan separat till `skills_omni/` av den privata förbättringspipelinen.

## 🧩 Core Concepts

Innan du jämför paket eller väljer en installationsväg hjälper det att förstå dessa fem byggstenar:

| Koncept | Vad det betyder |
|:--------|:------------|
| 🧠**Färdigheter**| Återanvändbara `SKILL.md`-spelböcker som lär en assistent hur man kör ett arbetsflöde väl |
| 📦**Katalogartefakter**| Genererade JSON- och arkivutgångar som möjliggör sökning, jämförelse, nedladdning och installation |
| 🔌**MCP Config**| Konfiguration på klientsidan för assistenter att upptäcka Awesome Omni Skills genom MCP-verktyg |
| 🤖**A2A Runtime**| Agent-till-agent-orkestrering för upptäckt, rekommendation och överlämnande av installationsplan |
| ✨**Curated Output**| `skills_omni/` — den Awesome Omni Skills-underhållna förbättrade ytan, skild från inhemskt uppströms intag |

>**📝 Inbyggd/kurerad policy:**
> - `skills/` accepterar ursprungligt uppströmsintag på vilket språk som helst, oavsett om det skickas direkt eller föreslås av den privata externa synkroniseringskörningen
> - `skills_omni/` är alltid kurerad och publicerad på engelska
> - `skills_omni/` är en enkelriktad yta och går inte tillbaka till naturligt intag
> - direkta bidragsgivare och förvarsbaserade externa intag delar samma validator, metadatauppdatering och förstärkarväg---

## 🛡️ Security and Release Posture

> Awesome Omni Skills levererar en starkare release- och verifieringshistoria än ett vanligt markdown-förråd.### 🧪 Validation and Smoke Checks

```bash
npm run validate         # Skill validation and metadata generation
npm run build            # Full build pipeline
npm test                 # Legacy integration tests + Vitest unit suite
npm run test:unit        # Fast Vitest unit tests only (~500ms)
npm run test:coverage    # Vitest with V8 coverage
npm run smoke            # Full release preflight
```

<detaljer>
<summary>📋 <strong>Vad pipelinen validerar</strong></summary>

- ✅ Kompetensvalidering och generering av metadata
- ✅ Kritisk säkerhetsgrind på native intake innan PR-fusion
- ✅ Verktyg för taxonominormalisering och omkategorisering
- ✅ Katalog- och arkivgenerering
- ✅ äldre integrationstester (Python PTY + Node TUI-påståenden)
- ✅ Vitest-enhetssvit för katalog-kärna poängsättning, sökning och filtrering logik
- ✅ API-, MCP- och A2A-startvägar
- ✅ Arkivverifiering
- ✅ OpenAPI 3.1-kontrakt serveras via Swagger UI på `/docs`
- ✅ Paketpreflight med `npm pack --dry-run`</details>

<detaljer>
<summary>🔐 <strong>Släpp hållning</strong></summary>

| Kontroll | Beskrivning |
|:--------|:--------|
| 🔒 SHA-256 kontrollsummor | Kontrollsummanifest för alla arkiv |
| ✍️ Signerade artefakter | Fristående signaturer på utgivningsartefakter |
| 🤖 CI-tillämpad | Releaseverifiering i CI före publicering |
| 🦠 Skannerportar | ClamAV och VirusTotal krävs båda i releaseverifiering innan publicering |
| 📦 GitHub Release | Automatiserad GitHub Release generation |
| 📋 npm-publikation | Endast från verifierad tarball |
| 🔄 Autofrigöring | Vid kvalificerad skicklighet smälter samman till `main` |

**Auto-release utlöses endast när en sammanslagning ändras:**
- `skills/*/**`
- `skills_omni/*/**`
- `data/bundles.json`

Ändringar av endast dokument**utlöser inte paketpublicering.</details>

<detaljer>
<summary>🛡️ <strong>Säkerhetskontrakt för förvar</strong></summary>

– Native intake accepterar inte färdigheter med kritiska säkerhetsfynd.
- Den statiska valideraren blockerar högriskmönster innan en färdighet kan smälta in i den offentliga katalogen.
- De nuvarande hårda blockerarna inkluderar:
  - fjärrinnehåll skickas direkt till skalexekveringen
  - instruktioner som försöker avslöja uppmaningar, hemligheter eller dold körtidskontext
- Publicerade releaser går igenom ett strängare andra lager:
  - ClamAV-skanning
  - VirusTotal hash-sökning
  - verifiering av arkivkontrollsumma
  - fristående signaturverifiering i release CI

Detta är en fördel för användaren, inte bara att bygga VVS: katalogen gynnar installationsbara färdigheter som redan passerat samma restriktiva säkerhetsgrind som projektet använder internt.</details>

## 🗂️ Repository Layout

| Väg | Syfte |
|:-----|:--------|
| 📂 `färdigheter/` | Kanoniskt författade färdigheter och inhemskt intag från direkta PR eller granskade förvarsbaserade importer |
| ✨ `skills_omni/` | Kurerade Omni-underhållna förbättrade derivat |
| 📖 `docs/` | Dokumentation för användare, bidragsgivare, arkitektur, drift och specifikationer |
| 📦 `dist/` | Genererade manifest, paket, kataloger och arkiv |
| 📁 `data/` | Buntdefinitioner och statiska stödjande data |
| 🧠 `paket/katalog-kärna/` | Delad katalogkörning med "ICatalogStorageAdapter" beroendeinjektion |
| 🌐 `paket/server-api/` | Skrivskyddat HTTP API med OpenAPI/Swagger UI på `/docs` |
| 🔌 `paket/server-mcp/` | MCP-server och lokal sidovagn |
| 🤖 `paket/server-a2a/` | A2A körtid och uppgift orkestrering |
| 🖥️ `paket/cli/` | Unified CLI-ingångspunkter, installationslogik och Ink Visual TUI (ESM-native) |
| ⚙️ `verktyg/skript/` | Validering, generering, release och testskript |
| 🧪 `vitest.workspace.js` | Vitest monorepo arbetsyta konfiguration |

>**ℹ️**`dist/` är avsiktligt versionerad eftersom de genererade artefakterna är en del av installations-, API-, MCP-, A2A-, rök- och releasekontraktet.---

## Bidra

Awesome Omni Skills accepterar inbyggt uppströms färdighetsintag under `skills/`.

| Regel | Detaljer |
|:-----|:--------|
| 📥 Inhemskt intag | Kan vara grov, författad på vilket språk som helst och kan komma från direkta PR eller granskade externa importgrenar |
| ✨ Kurerad utgång | `skills_omni/` reserverad för automationsförfattade Omni-derivat |
| 🌍 Uppströms repoförslag | Lägg till eller uppdatera en rad i [REPOSITORY-SOURCES.md](REPOSITORY-SOURCES.md) för att nominera ett arkiv för operatörsgranskning; repo-first-förslag kan nu använda `branch=auto` och `skills_path=auto` |
| 🚫 Manuella redigeringar | Offentliga manuella redigeringar av `skills_omni/` avvisas |
| 🔄 Upparbetning | Privat förstärkare omarbetar inhemska förändringar och uppdaterar utvald baslinje |

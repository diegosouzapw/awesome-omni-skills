Use this as a model for a narrow AI SDR pilot.

## Pilot goal

Validate whether AI-assisted qualification and reply routing can increase qualified meetings without harming sender health or meeting quality.

## Scope

- one ICP segment
- one geography or clearly reviewed region set
- one primary channel at first
- approved templates only
- human review for sensitive replies

## Week 0: readiness check

Complete before launch:

- sender readiness review
- compliance decision review
- qualification matrix drafted
- CRM owners defined
- handoff SLA defined
- pause conditions documented

## Week 1: limited launch

- start with narrow audience slice
- monitor sender health daily
- review all positive and compliance-sensitive replies manually
- record classification errors

## Week 2: evaluation review

- compare qualified meetings vs accepted meetings
- review rep feedback
- review false positives
- review handoff speed

## Go / no-go questions

- Did sender health remain stable?
- Did positive replies convert into acceptable meetings?
- Were opt-outs and objections handled correctly?
- Did CRM ownership remain clear?
- Were any unsafe claims made?

## Expand only if

- quality is stable
- safety boundaries held
- human follow-up is meeting SLA
- reps trust the routed output

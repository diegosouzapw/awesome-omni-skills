```md
# RFC: <clear action-oriented title>

- Status: Draft
- Owner / Driver: <name or team>
- Reviewers / Approvers: <names or roles>
- Created: <YYYY-MM-DD>
- Last Updated: <YYYY-MM-DD>
- Target Decision Date: <YYYY-MM-DD or TBD>
- Impact: <Low | Medium | High>
- Related Docs / Issues: <links or identifiers>
- Supersedes: <optional>
- Superseded By: <optional>

## Summary

<2-5 sentences summarizing the problem, options, and current recommendation or review state>

## Background / Context

- Current state:
- Why this matters now:
- Cost of inaction:
- Affected users / teams / systems:

## Problem Statement

<what decision must be made>

## Goals

- <goal 1>
- <goal 2>

## Non-Goals

- <non-goal 1>
- <non-goal 2>

## Stakeholders

- Driver:
- Approvers:
- Contributors:
- Informed:

## Assumptions

| Assumption | Confidence | Invalidation trigger |
| --- | --- | --- |
| <assumption> | <Low/Medium/High> | <what would make it false> |

## Constraints

- <budget, timing, platform, regulatory, staffing, or contractual constraints>

## Evidence / Data

- <incidents, user research, benchmarks, cost signals, audit findings, prior attempts>

## Decision Criteria

| Criterion | Weight | Type | Notes |
| --- | --- | --- | --- |
| <criterion> | <1-5 or %> | Must-have / Weighted | <why it matters> |

## Options Considered

### Option 1: <status quo or alternative>

- Description:
- Benefits:
- Drawbacks:
- Cost / Effort:
- Risks:
- Dependencies:

### Option 2: <alternative>

- Description:
- Benefits:
- Drawbacks:
- Cost / Effort:
- Risks:

### Option 3: <optional alternative>

- Description:
- Benefits:
- Drawbacks:
- Cost / Effort:
- Risks:
- Dependencies:

## Option Evaluation

<link or embed a decision matrix if helpful>

## Recommendation / Decision State

- Current recommendation:
- Why this option best meets the criteria:
- Why other options were not chosen:
- Decision status: <open / pending review / recommended / approved / rejected>

## Risks and Mitigations

| Risk | Severity | Mitigation |
| --- | --- | --- |
| <risk> | <Low/Medium/High> | <mitigation> |

## Success Metrics

- <metric 1 and target>
- <metric 2 and target>
- Review date: <date>

## Rollout / Migration / Rollback Considerations

- Rollout shape:
- Migration impact:
- Rollback expectation:

## Open Questions

- <question 1>
- <question 2>

## Action Items / Follow-ups

- [ ] <task>
- [ ] <task>

## Outcome

<leave as placeholder during draft; update after decision>

## References

- <link or source>
- <link or source>
```

## Notes

- Keep implementation details only at the level needed to evaluate the decision.
- Include the status quo as an option for significant changes.
- If the RFC intentionally uses normative requirement language, define that choice explicitly; otherwise prefer plain language.

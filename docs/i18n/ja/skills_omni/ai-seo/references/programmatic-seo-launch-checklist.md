Use this before publishing a new page set.

## Preflight

- [ ] The page pattern has unique value beyond entity swaps
- [ ] A pilot cohort of 3-10 pages is selected first
- [ ] Each page includes data, logic, or content variation that materially changes usefulness
- [ ] Title templates are specific and not repetitive boilerplate
- [ ] Meta descriptions, if written, match page intent

## Indexability and crawl controls

- [ ] Pages are included in XML sitemaps
- [ ] Canonicals are defined correctly
- [ ] Robots rules do not block intended pages
- [ ] Filter, faceted, or parameter URLs are controlled
- [ ] No accidental duplicate URL paths are being generated

## Internal linking

- [ ] New pages are linked from parent or hub pages
- [ ] Related pages link across the cluster where useful
- [ ] No important pages are orphaned

## Page quality

- [ ] Each page has a clear answer near the top
- [ ] Key sections are self-contained and scannable
- [ ] Tables, examples, or structured comparisons are used where helpful
- [ ] Claims are sourced or flagged for verification

## Technical spot checks

- [ ] A sample of pages is checked for title rendering
- [ ] A sample of pages is checked for canonical output
- [ ] A sample of pages is checked for schema matching visible content
- [ ] A sample of pages is checked for acceptable page experience

## Rollout

- [ ] Search Console is ready for weekly inspection
- [ ] Success and failure thresholds are defined
- [ ] Expansion is blocked until the pilot cohort validates

## Optional note on IndexNow

IndexNow may help with participating search engines, but do not treat it as a Google indexing guarantee.
